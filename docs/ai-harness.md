# Agent 推荐 Harness

处理链固定为：`用户输入 → 长度/风险检查 → 候选提取 → Schema 与证据验证 → Acceptance Policy → 固定画像评分 → 推荐结果`。

状态只有 `idle`、`needs_clarification`、`recommended`、`rejected`。只有确定性代码能产生最终状态和 Agent 排序；Provider 不能写入状态、分数、产品资料或最终赢家。

评分使用版本化的 10 个产品画像：任务匹配权重最高，其次为运行环境、自主程度与权限偏好。否定约束会扣分；相同输入与规则版本必须得到相同顺序。页面只展示匹配理由和边界提醒，不展示综合分，也不宣称绝对最佳。

Trace 仅包含规则版本、输入长度、候选数量、接受字段和拒绝数量，不记录完整原文、提示词或账户信息。Provider 超时、异常或非法输出时，系统回退到纯规则 Provider；仍无法识别当前任务时进入澄清，原有对比选择保持不变。
